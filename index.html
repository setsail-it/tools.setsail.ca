Workflow Guide
Interface

Left sidebar — shows all 11 pipeline stages. Green checkmark = done. Active stage highlighted dark. Locked stages (grey) require completing earlier steps first. Click any accessible stage to jump directly.

Top bar centre — AI bar — three modes: (1) Generating: indeterminate pulse + elapsed timer while a Claude call is running. (2) Notification: success or error message after an action (hover to see full text if truncated). (3) Tips: stage-specific workflow tips from this guide rotate every 7s when idle — pauses during generation, resumes after.

Project switcher — click the project name in the top bar to switch between projects without going back to the projects screen.

Admin panel — click your name (top right) → Admin. Manage team members: name, email, role (Admin / Strategist / Viewer), and project access (specific project IDs or blank = all). Edit your own name from the same panel. Log out via the same dropdown.

Button tooltips — hover any action button for 0.3s to see a description of exactly what it does. Implemented across Keywords, Briefs, and Sitemap stages.

Pipeline Overview
S1Setup — client name, URL, geo, voice
S2Site Snapshot — domain authority, link metrics
S3Research — upload docs, AI enriches 130 fields
S4Keywords — seeds → expand → volumes → opportunities
S5Sitemap — page architecture + keyword mapping
S6Briefs — enrich → assign → generate → evaluate
S7Copy — generate → audit → QC → approve
S8Images — generate per page
S9Layout — assign components per page
S10Schema — FAQ + structured data generation
S11Export — package copy, images, schema + layout for handoff
Stage 1 — Setup
·Client name, website URL, geo, and voice — these feed into every prompt throughout the pipeline. Fill them carefully before proceeding.
·Geo text drives country auto-detection for keyword volumes. If geo says "Vancouver, BC" the system targets Canada. If "Sydney" → Australia. You can override this manually in the Keywords stage.
Stage 2 — Site Snapshot
·Runs DataForSEO domain metrics — Domain Rating, referring domains, organic traffic estimate, top pages.
·Backlinks API is intentionally disabled (cost decision). DR and referring domain counts still pull correctly from the domain metrics endpoint.
·The DR from this snapshot feeds into the Site DR field in the Keywords Opportunities tab for rankability scoring. Copy it across manually.
·Snapshot data is stored in S.snapshot and saves with the project. Re-run only if domain metrics have changed significantly.
Stage 3 — Research · Best Order

Pages are grouped and ordered: Recently Edited (top 3) → Core → Services → Industry → Locations → Blog → Other. Within each group: P1 → P2 → P3. This order is consistent across Briefs, Copy, Images, and Layout. Editing or approving a brief moves the page into the Recently Edited group.

  1. Upload client documents — strategy doc, intake form, existing content, or anything with context. Claude extracts ~130 structured fields across 6 tabs.
  2. Review each tab — Business, Audience, Brand, Schema & Local, Competitors, Keywords. Correct anything Claude got wrong.
  3. Re-Enrich if needed — the Re-enrich button bypasses the skip logic and forces Claude to overwrite all fields, even pre-filled ones.
  4. Competitor list is the most important output here — it feeds SERP Intel and the sitemap competitor keyword mining in Stage 4.
⚠ Do not pre-fill default values in research fields before enrichment — the skip logic treats any non-empty string as "already filled" and won't overwrite it.
Stage 4 — Keywords · Best Order
  1. Set Country — the dropdown left of Fetch Volumes auto-detects from your geo text (AU/CA/US/GB/NZ/SG/ZA). Override manually if wrong — it saves with the project. Wrong country = wrong volumes.
  2. AI Generate Seeds — produces 20–30 tight head terms (2–4 words). These are for volume lookup, not long-tail guesses. Add competitor URLs in the Competitor Keywords section for additional seed mining.
  3. Copy seeds (📋 icon in Seeds table header) to paste into Ahrefs for cross-validation before fetching volumes.
  4. Seed bucket filter bar — the Seeds table header shows four source buckets: Mechanical (built from client/service/geo data), AI, Competitor, and Questions. Each shows a count or — if empty. Click any bucket to filter the list to that source. Verify all relevant buckets are populated before Fetch Volumes.
  5. Mechanical Seeds button — rebuilds the Mechanical bucket from current Setup data (client name, services, location) and merges back in. Run after any Setup changes. Does not touch AI or Competitor seeds.
  6. Pull from Questions — extracts the core 2–7 word keyword phrase from each question (strips "how much does", "what is", etc.) and adds to Seeds. Full questions won't return DataForSEO volumes — only the extracted phrase will. The Questions bucket in the filter bar shows how many were extracted.
  7. Fetch Volumes — Google Suggest expands each head term into 400–600 real phrases, then DataForSEO volumes each one. Top 300 by score are kept. This is your full keyword pool.
  8. Enter Site DR in the Opportunities tab top-right — this adjusts the rankability indicator per keyword. Use the DR from Stage 2 Snapshot.
  9. Review Opportunities tab — keywords scored by: log(vol+1) × 100 / max(KD, 5). Minimum vol: 10. Keywords showing "—" have no DataForSEO data for that country — you can still manually select them.
  10. Select Top 50 then manually add any low-volume but strategically important terms (e.g. high-CPC, direct buyer intent).
  11. Cluster Selected — groups keywords into topical clusters for sitemap page creation.
  12. Questions tab — pure bottom-of-funnel questions only: pricing, ROI proof, risk/vetting, agency comparisons, local buying intent. Not educational — these are what a business owner searches when they’ve decided they need help and are now vetting agencies. Validate & Assign checks DataForSEO volumes and maps each question to a page (vol is a signal not a gate — zero-vol questions still feed FAQ schema and copy). + More Questions appends another 20 BOF questions in the same intent categories, deduping automatically. Intent badge uses AI assignment type post-validation. Coverage column shows once clusters exist — gap questions get a “+ Sitemap” button to queue a page directly from this table.
  13. Seeds tab ← Pull from Questions — converts all questions to seed keyword form and appends to Seeds. Run after More Questions to keep the keyword pool fed without switching tabs.
⚠ Run Fetch Volumes before building the Sitemap. Pages won't have volume data otherwise and the sitemap generator won't be able to justify page creation thresholds.
Stage 5 — Sitemap · Page Rules
Two types of pages — Claude enforces volume thresholds for SEO pages:
StructuralHomepage, /about, /contact, /case-studies — always built, vol:0 acceptable
/services/vol > 0 required — any keyword cluster justifies creation
/industries/vol > 100 required
/locations/vol > 100 required
/blog/vol > 50 required
Keyword intent rules (enforced in prompts):
·Homepage — brand/category only: "Vancouver marketing agency". NEVER specific service terms.
·Service pages — service-specific only: "SEO services Vancouver", "PPC agency BC".
·Location pages — geo + category: "digital marketing Calgary", "marketing agency Toronto".
  1. Generate Sitemap — Claude maps pages to keyword clusters. Review the output carefully.
  2. Display order — pages are automatically sorted for review: Core (home, about, contact) → Services → Locations → Blog. Within each type: P1 → P2 → P3 → score. This is display-only — it does not change S.pages order or generation priority.
  3. Category tabs — filter by All, Services, Locations, Blog, or Core. Counts show live as pages are added or removed.
  4. Reorder pages — use edit mode to adjust page types, priorities, or slugs. P1 pages are processed first throughout the pipeline.
  5. Run Keyword Mapping — assigns primary + supporting keywords to each page from the opportunity pool.
  6. Approve Sitemap — locks the page architecture. Required before Briefs stage processes.
Stage 6 — Briefs · Best Order
  1. Expand Niche Keywords — runs Google Suggest on each page's primary keyword with 10 modifiers. Pulls niche variants and gets DataForSEO volumes. Tagged by page slug so AI Assign knows which keywords belong where. Run globally from the toolbar, or per-page via the ⌖ Niche KW button on each brief card.
  2. Generate Page Questions — one Claude call generates 8 targeted FAQs per page based on primary keyword + page type. Auto-assigned to each page's assignedQuestions[]. Run globally or per-page via the ? Questions button on each brief card.
  3. AI Assign Keywords + Questions — distributes remaining global PAA questions and keyword pool across pages. Skips already-assigned page questions (no duplicates). Run globally from the toolbar. The per-page ✦ Assign button on each brief card runs a full curation pass: Claude reviews what's currently assigned, keeps what matches the page's intent, drops what doesn't, and replaces it with better fits from the pool. The button shows a removal count (e.g. ✓ Curated (3 removed)) when items were dropped.
  4. SERP Intel (per page, or auto-runs at brief generation) — fetches top 3 organic results for the primary keyword, extracts: word count, H2 structure, KW density. Sets word count target at max competitor × 1.05. Some sites block crawlers — this is normal.
  5. Generate / Regenerate Brief — click any page row to expand it (accordion). Once open, use the top-right action bar to generate, regenerate, or run a new version. SERP Intel auto-runs silently if not already done.
  6. Queue All — submits all un-generated briefs to the Cloudflare Queue for server-side generation. Tab-close safe — the worker generates briefs in the background even if you leave. Processes 2 concurrent, retries automatically on rate limits. Progress shows in the AI bar with estimated time remaining. Results merge back into the page when done.
  7. Evaluate Brief — AI scorecard: 9–11 checks including reader profile, H1, CTA presence, objections, trust signals, SERP word count target, H2 coverage.
  8. Version pills (V1 / V2…) — appear in the score bar once more than one draft exists. Click to switch between versions. Last 2 kept — V1 is dropped when V3 is pushed. Run V2 via + V2 button in the action bar.
  9. Approve Brief — button in the top-right action bar of each card. Locks the brief. Copy stage will not process a page without an approved brief.
✓ Steps 1–3 take about 3 minutes total and dramatically improve brief specificity — especially for niche service/industry pages that previously got generic questions and thin keyword coverage.
Stage 7 — Copy · Best Order
  1. Brief must be approved before copy generates for that page. The Generate button is locked otherwise.
  2. Generate Copy (Pass 1) — full page HTML from brief. Stored as draft v1. Copy HTML is saved to its own KV key (not the main project payload) so it survives page reload.
  3. Run Audit — deterministic checks first (keyword density 0.3–3.0%, SERP-calibrated ceiling if SERP Intel ran), then AI checks: H1 contains keyword, primary keyword in intro, intent match, no placeholders, word count vs target, Canadian spelling, no AI fluff. Page-type specific checks added on top.
  4. Pass 2 — Claude surgically rewrites weak sections based on audit findings. Generates a new draft tab without replacing v1.
  5. Human QC — 7 items per page (6 universal + 1 page-type specific):
    • Stats & numbers verified (every %, $, count confirmed against real source)
    • Client proof approved for use (named clients/logos have permission)
    • Offer & pricing current (guarantees match current offering)
    • CTAs & links work (no 404s or dead forms)
    • Brand voice on-point (sounds like us, not generic AI)
    • No stale content (no outdated team refs, discontinued services)
    • + 1 page-type specific item
  6. Approve Copy — locks the page for export and Schema generation. Last 2 copy drafts kept per page.
⚠ Draft version tabs (v1/v2/v3) are in-memory only. Only the active draft persists across a page reload. Version history is lost on refresh — do not rely on it for long-term storage.
Key Limits & Architecture Notes
·Keyword pool cap: top 300 scored, shown all in Opportunities. Vol "—" = no DataForSEO data for that country — still selectable manually.
·Keyword scoring: log(vol+1) × 100 / max(KD, 5). Threshold: vol ≥ 10 to score above 0. KD of 0 is treated as 5 to avoid divide-by-zero.
·Country selector auto-detects from geo text but saves your manual selection with the project. Always verify before Fetch Volumes.
·SERP Intel: fetches top 3 organic results. Some sites block crawlers (fetch_ok: false) — this is normal. Directives still generated from whatever was fetched.
·Brief versions: last 2 kept. V1 is dropped when V3 is generated. Regenerate Brief = update current draft in-place. Run V2 = push new draft.
·Copy HTML is stored in separate KV keys per page (not in the main project payload) — survives reload. Main payload stores only audit results and approval state.
·Images — base64 image data stripped from save payload (too large). Only prompt + status saved. Images must be regenerated on a new session.
·Monthly keyword trends stripped from save payload — reconstructed from DataForSEO on next Fetch Volumes run.
·SERP Intel competitors stripped from save payload — only directives + metadata persists. SERP competitors visible in brief card only until next save.
·KV limit: 25MB max. Monitor console for [save] payload size warnings. Projects with 100+ pages and heavy research data approach this limit.
·Save: auto-saves on every action (debounced). Green indicator top-right confirms saved. Red = failed — check console for payload size or network error.
·Auth: Cloudflare Access (Google SSO). Any @setsail.ca email can authenticate. Projects are user-scoped (u:{email}:project:{id}). Roles (Admin/Strategist/Viewer) and per-project access restrictions managed in Admin panel (top-right dropdown → Admin).
·Token limits: Default 3k. Briefs: 4k. Copy: 6k. Schema: 4k. Research: 3.5k. Gap analysis: 3k.
·Bulk queue: Cloudflare Queues (setsailos-gen-queue). 2 concurrent, 3 retries, 30s delay on 429. Job status in KV, 24h TTL. Free tier handles ~10k ops/day — 40 briefs ≈ 120 ops.
·Canadian spelling: -ize endings are correct (optimize, organize, specialize). Only flag: color/honour/center/theatre and -yze → -yse (analyze → analyse).
